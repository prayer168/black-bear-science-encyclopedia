# CLAUDE.md — 黑熊老師自然科學繪本百科：AI 協作規範

本文件是 AI（Claude）在本專案中產製與維護內容時必須遵守的規範。

## 專案是什麼

《黑熊老師自然科學繪本百科》：國小三～六年級自然科學「科普繪本百科」靜態網站，
可部署 GitHub Pages，全站**不使用外部 CDN**、不需建置工具，純 HTML/CSS/JS。

課程參考來源：https://prayer168.github.io/science-course/ （115學年度康軒／翰林／南一課程架構）。
只作為「主題庫與銜接參考」——**禁止照抄課本單元順序與文字**，必須轉化為百科主題。

## 關鍵檔案

- `data/topics.json`：40 篇主題地圖的**正式資料來源**（id、標題、年級、分類、核心概念、國中銜接、狀態、路徑）。
- `assets/js/main.js`：開頭有 `const TOPICS = [...]`，是 topics.json 的**同步副本**（因 file:// 無法 fetch）。
  ⚠️ 修改主題清單時，兩處必須同步更新。
- `data/grade-map.json`：年級定位與國中銜接對照。
- `data/glossary.json`：詞彙表。
- `templates/`：所有新文章必須從 `article-template.html` 複製起手，內容結構依 `article-template.md`。

## 新增一篇文章的流程

1. 在 `data/topics.json` 找到該主題（或新增），確認 id、年級、分類。
2. 複製 `templates/article-template.html` 到 `topics/grade-N/<id>.html`。
3. 依 14 區塊結構寫滿內容（缺一不可，清單見下）。
4. 用 `templates/factcheck-template.md` 逐項自我查核，結果寫進文章第 14 區。
5. 更新 `data/topics.json` 該篇 `status` 為 `published`、填入 `path`。
6. 同步更新 `assets/js/main.js` 的 TOPICS。
7. 若該分類索引頁存在，加上連結。

## 每篇文章 14 區塊（缺一不可）

1. 標題（清楚、有趣、有科普感）
2. 基本資料卡（適合年級／可銜接年級／主題分類／核心概念／關鍵詞／國中銜接概念／建議閱讀時間／搭配課堂活動）
3. 一句話吸引學生
4. 繪本式開場故事：300–500 字，有情境、角色與問題，**不可直接定義概念**
5. 角色對話：≥8 輪，每輪都要推進科學理解，幽默但不幼稚
6. 核心概念三層：小學生好懂版／高年級進階版／國中銜接版
7. 生活中的例子：3–5 個，貼近校園、家庭、戶外
8. 圖解設計 ≥3 張（概念總覽圖／過程機制圖／實驗或應用圖），每張含：圖名、目的、應出現元素、必標科學概念、圖說、AI 繪圖提示詞、避免畫錯提醒
9. 繪本分鏡 ≥6 頁，每頁含：頁碼、畫面描述、旁白、角色台詞、科學概念、圖像提示詞、不能畫錯的地方
10. 小實驗 ≥1：活動名稱、適合年級、材料、步驟、**操縱變因、應變變因、控制變因**、觀察紀錄表、預期結果、科學解釋、安全提醒
11. 迷思破解 ≥5 項：迷思說法／為何易誤解／正確說法／怎麼驗證
12. 小測驗：5 選擇＋3 簡答＋2 探究思考，每題附答案與解析
13. 給老師的教學補充：教學目標、對應學習重點、引導問題、常見迷思、差異化建議、延伸活動、評量建議、Rubric
14. 內容查核表（8 項檢核）

## 角色聲音指南（寫對話時遵守）

- 🐻 **黑熊老師**：溫暖、幽默、會用比喻，負責總結與破迷思。不說教。
- 🦊 **小狐同學**：問「學生真的會問的問題」，包含天真的錯誤直覺。
- 🪲 **甲蟲博士**：補充冷知識與科學史，一次只補一個點，不艱深。
- 💧 **水滴小隊長**：只在實驗／觀察情境出現，口頭禪是提醒「控制變因」與安全。
- 👾 **迷思怪獸**：理直氣壯地講錯，語氣可愛不討厭，被破解後會心服口服。

## 科學正確性紅線

1. 不確定就標示「⚠️ 需要查證」，**絕不編造**數據、年代、人名。
2. 比喻可以簡化，但不能造成錯誤心智模型（例：不可說「電子像水一樣被用光」）。
3. 清楚區分觀察（看到什麼）、推論（想到什麼）、解釋（為什麼）。
4. 常見必守事實：昆蟲成蟲六足三體段；光合作用主要在葉綠體、需光/水/CO₂、產生葡萄糖與氧；熱傳導/對流/輻射不可混用；食物鏈箭頭方向是「能量流向」（被吃者→吃它者）；蜘蛛不是昆蟲。
5. 實驗必須安全、低成本：不得涉及明火無成人陪同、危險化學品、玻璃碎裂風險高的操作。

## 網頁技術規範

- RWD：手機、平板、電腦、教室大螢幕（大字模式）。
- 全站共用 `assets/css/style.css` 與 `assets/js/main.js`，文章頁以相對路徑 `../../assets/...` 引用。
- 教師補充區、內容查核區用 `<details>` 摺疊；測驗答案用 `<details class="answer">`。
- 對話泡泡 class：`.bubble.bear / .fox / .beetle / .drop / .monster`。
- 圖片一律先用 `.img-placeholder` 佔位區（內含圖名與提示詞摘要），之後再換真圖。
- 不使用任何外部 CDN、Google Fonts、外部框架。
- 語言：`lang="zh-Hant"`，用字遵循台灣國小自然科慣用語（例：光合作用、蒸散、變態、導體）。

## 命名規範

- 文章檔名：小寫英文 kebab-case，與 topics.json 的 `id` 一致，例：`plants-need-sunlight.html`。
- 分類代碼固定九個：plants-life, animals-insects, air-water-weather, sound-light-electricity,
  force-motion-machines, matter-heat-change, earth-resources, ecology-sustainability, inquiry-methods。

## 完成後自我檢查

每次產出文章後，必須列出：已完成項目清單、待補強項目清單（含「需要查證」項目）。
